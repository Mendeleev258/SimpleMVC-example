# Инструкция по исправлению кодировки русских символов

## Проблема
Русские символы отображаются неправильно на веб-страницах (например, "ÐŸÐµÑ€Ð²Ð¾Ð¿Ñ€Ð¾Ñ…Ð¾Ð´Ñ†Ñ‹" вместо "Первопроходцы"), несмотря на правильное хранение в базе данных.

## Причина
Данные были изначально загружены в базу данных с неправильной кодировкой, и хотя в базе они теперь отображаются правильно, при передаче в веб-приложение возникает проблема с кодировкой.

## Решение
Для полного исправления проблемы выполните следующие шаги:

### 1. Остановите контейнеры
```bash
docker-compose down
```

### 2. Удалите данные базы данных
```bash
docker volume rm simplemvc-example_mysql_data
# или если имя тома другое, используйте:
docker volume ls
# чтобы посмотреть точное имя тома и удалить его
```

### 3. Запустите контейнеры заново
```bash
docker-compose up -d
```

После выполнения этих действий база данных будет создана заново с правильной кодировкой, и данные из dump файла будут загружены с правильной кодировкой.

## Проверка результата
После перезапуска откройте веб-страницу и проверьте отображение русских символов.

2. Добавлены команды установки кодировки в файл dump/create_all_tables.sql

4. Обновлена Apache конфигурация
